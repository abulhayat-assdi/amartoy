A. Overall Design System 
This section defines every visual token that governs the entire AmarToy front-end. Every color, font, spacing 
value, shadow, and shape rule documented here must be codified as CSS custom properties (variables) in a single 
:root block so that every component automatically inherits them. No magic numbers should ever appear directly 
inside component CSS.
 
 
A.1 ΓÇö Color Palette 
The AmarToy color language is warm, playful, and high-contrast. It uses an orange-dominant primary with a 
warm yellow secondary, which immediately communicates fun and energy to its target audience of parents and 
children. The site background is a warm off-white rather than pure white, which softens the brightness and feels 
more premium.

B. Global Component Architecture 
These components appear on every page and define the outer shell of the application. They must be coded as 
isolated, reusable components ΓÇö in React they are top-level layout components; in plain HTML they are included 
via partials or template literals.
 
 
B.1 ΓÇö Header / Navigation Bar 
Visual Anatomy 
The header is a sticky, full-width bar that transitions from transparent (overlaid on the hero) to a solid 
white/opaque state when the user scrolls down past 80px. It has three distinct zones:
 
ΓùÅ 
LEFT ZONE (25% width): Logo image. Links to home. On mobile, this also serves as the brand anchor when 
the hamburger is tapped.
 
ΓùÅ 
CENTER ZONE (50% width): Horizontal navigation menu. Items: Home (dropdown), Pages (dropdown), 
Shop (dropdown), Blog (dropdown), Contacts. Each top-level item has a dropdown submenu that appears 
on hover.
 
ΓùÅ 
RIGHT ZONE (25% width): Three icon controls ΓÇö (1) Cart icon with item-count badge bubble, (2) Search 
icon (opens a full-width search overlay), (3) Hamburger/Panel icon (opens the right-side info panel).

D. Animation & Interaction Guide 
All animations on the AmarToy site serve a single purpose: to make the experience feel playful and alive, 
consistent with a toy brand. They are generally not long or aggressive ΓÇö quick, springy, and delightful. Below is a 
complete catalog of every animation type and the exact CSS/JS needed to reproduce it.
 
 
D.1 ΓÇö Scroll-Triggered Reveal Animations 
Elements animate in as they enter the viewport. Use the Intersection Observer API to add an .animate-in class 
when an element reaches the viewport. All animatable elements start with opacity: 0 and a translateY offset.
 
 
Base Utility Classes 
/* All elements that animate should have this class by default */
 
.reveal { opacity: 0; transform: translateY(40px); transition: opacity 0.6s ease, 
transform 0.6s ease; }
 
.reveal.visible { opacity: 1; transform: translateY(0); }
 
 
/* Staggered children ΓÇö add delay via CSS custom property */
 
.reveal-delay-1 { transition-delay: 0.1s; }
 
.reveal-delay-2 { transition-delay: 0.2s; }
 
.reveal-delay-3 { transition-delay: 0.3s; }
 
.reveal-delay-4 { transition-delay: 0.4s; }
 
 
Intersection Observer JS 
const revealObserver = new IntersectionObserver((entries) => {
 
  entries.forEach(entry => {
 
    if (entry.isIntersecting) {
 

      entry.target.classList.add('visible');
 
      revealObserver.unobserve(entry.target); // animate once
 
    }
 
  });
 
}, { threshold: 0.15 });
 
 
document.querySelectorAll('.reveal').forEach(el => revealObserver.observe(el));

D.6 ΓÇö Number Counter Animation (Stats Section) 
In the About section, stat numbers (e.g., "5000+ Happy Kids") animate from 0 to their target value when scrolled 
into view.
 
function animateCounter(el) {
 
  const target = parseInt(el.dataset.target);
 

  const duration = 1800;
 
  const start = performance.now();
 
  function update(time) {
 
    const progress = Math.min((time - start) / duration, 1);
 
    const ease = 1 - Math.pow(1 - progress, 3); // ease-out cubic
 
    el.textContent = Math.floor(ease * target).toLocaleString() + '+';
 
    if (progress < 1) requestAnimationFrame(update);
 
  }
 
  requestAnimationFrame(update);
 
}

E.4 ΓÇö Performance Considerations 
ΓùÅ 
Use next/image (or native lazy loading) for all product and blog images to avoid layout shift. 
ΓùÅ 
Load Swiper.js only on pages that use it (dynamic import in Next.js). 
ΓùÅ 
CSS animations use transform and opacity only ΓÇö never animate layout properties (width, height, top, left) 
for 60fps performance.
 
ΓùÅ 
Use will-change: transform on elements that animate on every frame (hero character, floating shapes). 
ΓùÅ 
Preconnect to Google Fonts: <link rel="preconnect" href="https://fonts.googleapis.com">. 
ΓùÅ 
Use font-display: swap to avoid invisible text during font load. 
 
E.5 ΓÇö Accessibility Checklist 
ΓùÅ 
All interactive elements have clear :focus-visible styles (orange outline with offset). 
ΓùÅ 
All images have descriptive alt attributes. 

ΓùÅ 
Navigation dropdowns are keyboard-accessible (arrow keys + Escape to close). 
ΓùÅ 
Side panel traps focus when open (focus-trap-react or manual tabindex management). 
ΓùÅ 
Cart count badge uses aria-label="X items in cart". 
ΓùÅ 
Wishlist heart uses aria-pressed="true/false" and aria-label. 
ΓùÅ 
Form inputs are connected to labels via htmlFor / for attributes (not just placeholder). 
ΓùÅ 
Color contrast for primary orange (#FF6B35) on white: approx 3.1:1 ΓÇö use for large text/icons only. For 
body text always use #1C1C1C on white (21:1 ratio).
